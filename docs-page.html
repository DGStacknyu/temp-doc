Introduction

Welcome to our project, a dynamic web application crafted with SvelteKit, Tailwind CSS, TypeScript, and various SvelteKit libraries. Leveraging SvelteKit's efficient compilation process and component-based architecture, we create highly performant user interfaces. Tailwind CSS ensures consistency and responsiveness, while TypeScript enhances code robustness and maintainability. Through the integration of SvelteKit libraries, from pre-built components to routing utilities and state management solutions, we deliver a modern, user-centric web application with exceptional performance and scalability.

This template uses Tailwind CSS for style:
Here is tailwindcss.config.js of project

      colors: {
        accent: "#ef6223",
      },
    },
   },
  plugins: [require("@tailwindcss/typography")],
};

Installation

Here are step-by-step instructions for users to download, set up, and run the SvelteKit project from the provided zip file:

1) Download the Zip File

Users should start by downloading the zip file containing the project code from the provided link or source.

2) Extract the Zip File

Once the zip file is downloaded, users should extract its contents to a location of their choice on their local machine. This will create a folder containing all the project files and directories.

3) Navigate to the Project Directory

Users need to open a terminal or command prompt and navigate to the directory where the project files are located. They can use the 'cd' command followed by the path to the project directory

4) Install Dependencies

Within the terminal or command prompt and inside the project directory, users should run the command to install the project dependencies. Typically, this is done using a package manager like npm or yarn. For npm, the command is 'npm install' and for yarn, 'yarn install'

5) Run the Development Server

After installing the dependencies, users can start the development server by running the appropriate command. In the terminal or command prompt and inside the project directory, they should execute 'npm run dev' or 'yarn dev' , depending on the package manager used. This command will compile the SvelteKit project and start a local development server.

6) Access the Application

Once the development server is running, users can access the SvelteKit application by opening a web browser and navigating to the specified URL, typically 'http://localhost:5173' or another address provided in the terminal output.

7) Explore and Modify

Users can now explore the SvelteKit project running locally in their web browser. They can interact with the application, view its functionality, and modify the code as needed using their preferred code editor.

8) Additional Steps (Optional)

Depending on the specific requirements of the project, users may need to configure environment variables, set up external services, or perform other tasks. Any additional steps or instructions should be provided in accompanying documentation or README files within the project folder..

By following these steps, users can easily download, set up, and run the provided SvelteKit project from the zip file, enabling them to explore its features and customize it to suit their needs.

Layout

This Svelte component represents a layout structure for the web application, including a header, footer, and a placeholder for the main content. It imports the header and footer components along with any necessary CSS files to style the layout. file located in src\routes\+layout.svelte

1) Header (Navbar)

The Hero section serves as the prominent introductory element of the webpage, designed to captivate the user's attention and convey key messaging effectively. Structured within a section tag, it adopts a responsive layout, ensuring optimal display across various screen sizes.

Accessing the Static Folder

Users should locate the static folder within the project directory. This folder typically contains assets such as images, fonts, and other static files that are served directly to the client.

Replacing Images

Users should locate the existing image files within the static folder, which are referenced in the <'img'> tags of the HTML code. They can then replace these image files with their desired images, ensuring that the new images have the same file names. After replacing the images, users should refresh the browser to see the updated images reflected in the Hero section.

Modifying Text Content

To change text displayed in the Hero section:

  1. Users can directly edit the HTML content within the <'p'> and <'h1'> tags to update the text displayed on the webpage.
  2. They should locate the corresponding HTML file containing the Hero section code and open it using a text editor.
  3. Within the HTML file, users can find the <'p'> and <'h1'> tags that contain the text content.
  4. By modifying the text within these tags, users can update the messaging to their desired content.
  5. After saving the changes to the HTML file, users should refresh the browser to see the updated text reflected in the Hero section.

Refreshing the Browser

1) After making changes to the static folder or HTML files, users should refresh the browser to ensure that the updated images and text are displayed correctly on the webpage.

2) They can do this by clicking the refresh button in the browser or by pressing the "Ctrl + R" or "Cmd + R" keyboard shortcut.

3) Our Services

This section of code represents the "Our Services" section of the webpage, showcasing various services offered. The content is structured to include a title, description, and service cards, with data sourced from a JSON file. Users can easily customize the text and service data by modifying the JSON file and images associated with each service.

Using the JSON Data File:

Users can find the data file named data.js located in the following directory: src\lib\data.js

To modify the service data:

  1. Open the 'data.js' file using a text editor.
  2. Each service is represented as an object within the 'services' array.
  3. Each service is represented as an object within the 'title','description','points','img','id' and 'icon properties of each service object to update the service information.
  4. Ensure that the 'img' property points to the correct image file path within the project's static folder.
Using Service Cards:
  1. Each service is displayed using a ServiceCard component.
  2. Users can modify the service title, description, and associated image by updating the corresponding properties passed to the ServiceCard component.
  3. The ServiceCard component is imported from the project's component directory, enabling easy customization of individual service cards.
Custom service section

The provided code represents a reusable modal component designed for users to contact service providers regarding a specific service offered on the website. This modal serves as a means for users to initiate communication with the service provider directly from the webpage, facilitating inquiries, consultations, or requests for additional information related to the particular service being showcased.

Customization:

  1. Users can customize the content of the custom service section by providing values for the 'title','description','img' and 'id' attributes when using the component.
  2. The form action URL action="https://formsubmit.co/" can be updated to direct form submissions to the desired endpoint.
  3. Styling classes and attributes can be modified or extended to match the project's design requirements.
  4. Users can include multiple instances of this component throughout their project, each with unique content and functionality.

4) Our Features Section

This section consists of two Svelte components used to showcase features on a webpage "FeatureCard" and "Title" These components are integrated within a section dedicated to highlighting the features offered by a service or product.

Feature Card Component :
  1. The FeatureCard component displays individual feature items, each represented by a card containing a title, an icon, and a description.
  2. It imports the Icon component from the '@iconify/svelte' library to render icons corresponding to each feature.
  3. Features are iterated over using the '#each' block, with data sourced from the 'features' array imported from the 'data.js' file.
  4. For each feature, the component renders a FeatureCard with properties for the title, icon name, and description.
Title Component:
  1. The 'Title' component is a reusable component imported from base/title.svelte , responsible for displaying a title at the top of the features section.
  2. It receives the title as a prop, allowing for dynamic insertion of the section title.
Customization:
  1. Users can customize the content of the features section by modifying the data in the 'features' array located in the 'data.js' file.
  2. Icons for each feature can be customized by updating the iconName property in the FeatureCard component or providing different icon names in the features data array.

5) Call to Action(CTA) Section

This section is a Svelte component responsible for displaying a limited-time offer section on a webpage. This section aims to attract users' attention with a compelling discount offer and encourage engagement through a countdown timer and a call-to-action button for initiating a chat.

Countdown Timer:
  1. The component initializes three variables 'hours','minutes' and 'seconds' representing the countdown time.
  2. The 'startCountdown()' function is invoked to decrement the time values every second until the countdown reaches zero.
  3. The countdown is displayed in a digital format with hours, minutes, and seconds using dynamic placeholders.
Content Section:
  1. The content section includes a heading with the limited-time offer message and a description highlighting the discount and urging users to take advantage of the offer.
  2. An image depicting a student is displayed alongside the offer content to add visual appeal and context to the section.
Customization:

To customize this component into a webpage

  1. Include the provided Svelte component code within the desired section of the webpage.
  2. Ensure that any necessary dependencies, such as the @iconify/svelte library, are installed and imported correctly.
  3. Customize the offer message, description, and chat button functionality as needed to align with the specific promotional campaign or business requirements.

6) Reviews (Testimonials) Section

This section is for for displaying reviews or testimonials on a webpage. It imports various components and libraries to manage the review display and styling.

Review Display:
  1. Reviews are displayed using the 'ReviewCard' component, which receives properties for the reviewer's name and their review content.
  2. The 'reviews' data is sourced from the data.js file located in the src/$lib/data.js directory.
  3. Reviews are iterated over using the '#each' block, and each review is rendered within a 'SplideSlide' component.
Splide Slider:
  1. Reviews are organized and displayed within a slider implemented using the 'Splide' component from the '@splidejs/svelte-splide' library 'npm @splidejs/svelte-splide'
  2. The slider offers options for customization, such as setting the number of slides per view, autoplay behavior, slide transition speed, and responsiveness for different screen sizes.
  3. Breakpoints are defined to adjust the number of slides displayed at different viewport widths, ensuring optimal viewing experience across devices.
Usage:

Steps to use this component into a webpage

  1. Include the provided Svelte component code within the desired section of the webpage.
  2. Ensure that the 'ReviewCard' component and 'reviews' data are appropriately imported and structured..
  3. Adjust the styling and customization options of the 'Splide' slider as needed to match the design and functionality requirements of the webpage.

By integrating this review section, website owners can effectively showcase positive feedback and testimonials from satisfied customers, enhancing credibility and trustworthiness, and potentially influencing other visitors' perceptions and decisions.

7) Contact Section

This Svelte component creates a contact section on a webpage where users can reach out to the website owner or service provider. It imports other components and data required for rendering the form and title.

Title Component:

The component imports the 'Title' component from the 'base' directory, which is responsible for displaying the section title. The title displayed is "Contact us."

Form Component:
  1. The component imports the 'Form' component, from the 'base' directory, which presumably contains the form structure and functionality for user input and submission.
  2. The 'servicesList' variable is imported from the 'data.js' which is located in src/$lib/data.js directory it contains data related to the services offered by the website or service provider.
Form Structure:
  1. The form includes fields for selecting the project type from a dropdown menu and specifying the deadline using a date input field.
  2. The 'servicesList' data is iterated over to populate the options in the project type dropdown menu.
Form Submission:
  1. Upon submission, the form data is sent to the specified action URL using the 'POST' method.
  2. Hidden input fields are included for additional form data, such as the next URL, subject, and other metadata.
  3. Users can also click on the "WhatsApp Us" button to open a link to WhatsApp with the provided phone number at the end of URL in code.

By integrating this contact section, website owners can facilitate communication with users or potential clients, allowing them to inquire about services, request quotes, or provide feedback conveniently through the provided form or WhatsApp link.
